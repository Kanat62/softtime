# SoftTime — МОБИЛЬНОЕ ПРИЛОЖЕНИЕ

# ТЕХНИЧЕСКОЕ ЗАДАНИЕ v2.0 (React Native — Июнь 2026)

> Документ описывает мобильное приложение. Тяжёлое управление (настройка расписаний всем, офисные сети, генерация QR, полные отчёты) — на вебе, в мобайле не дублируется.

**Без авторизации:** Splash → Onboarding / Login

**WORKER — нижняя панель (5 вкладок):**

```
Главная     |   Новости   |   Заявки   |   Профиль
```

**ADMIN — нижняя панель (5 вкладок):**

```
Главная   |   Новости   |   Заявки   |   Профиль тут внутри упавление и так далее
```

**Гарды навигации:**

- Статус `BLOCKED` → экран блокировки
- Компания `SUSPENDED` → приход/уход заблокированы, остальное доступно

## 5. МОДУЛЬ — АУТЕНТИФИКАЦИЯ

### 5.1 Splash

- Лого по центру на фоне `#1877F2`, плавное появление
- Silent refresh токена → Главная / Login

### 5.2 Onboarding (Стартовый экран)

- Два варианта: **«Войти»** и **«Зарегистрироваться»**
- Минималистично: лого, краткий слоган, две кнопки
- «Зарегистрироваться» → экран выбора роли

### 5.3 Выбор роли при регистрации

Две карточки на выбор:

```
┌─────────────────────┐   ┌─────────────────────┐
│  🏢  Я администратор │   │  👤  Я сотрудник     │
│  Создаю компанию    │   │  Есть код компании  │
└─────────────────────┘   └─────────────────────┘
```

### 5.4 Регистрация ADMIN (создание компании)

Поля (одна страница, без лишних шагов):

- Название компании
- ФИО
- Email
- Пароль (минимум 8 символов, показать/скрыть)

После создания → показать **код компании (6 символов)** на экране успеха с кнопкой «Скопировать» и пояснением «Передайте код сотрудникам для регистрации».

### 5.5 Регистрация WORKER

Поля:

- ФИО
- Email
- Пароль (минимум 8 символов)
- Код компании (6 символов) — отдельное поле с крупным шрифтом, разбивка визуально (типа OTP-поля)

После отправки → экран ожидания: «Заявка отправлена. Ждите подтверждения администратора» + иконка часов.

### 5.6 Вход (Login)

- Email + пароль
- Кнопка «Войти»
- Ошибки: неверные данные / заблокирован / ожидает подтверждения
- Ссылка «Зарегистрироваться» внизу

### 5.7 Токены

- Access 15 мин, Refresh 30 дней, оба в Secure Storage
- Перехватчик: 401 → silent refresh → при провале → logout + Login
- Logout → инвалидация refresh на backend + удаление FCM-токена

## 6. ГЛАВНЫЙ ЭКРАН (Home)

### Структура (сверху вниз):

**1. Шапка**

- Слева: приветствие («Привет [Имя] 👋») + текущая дата
- Справа: аватар (тап → профиль)

**2. Блок посещаемости**

- Карточка: «Посещаемость»
- Строка: «График: 10:15 – 18:00» (сегодняшнее расписание)
- Два мини-индикатора: Приход `-:--` / Уход `-:--` (заполняются после сканирования)
- Одна активная кнопка: **«Приход»** (зелёная) если не отметился, **«Уход»** (синяя) если уже пришёл
- Неактивная кнопка показывается, но с opacity 0.4 и недоступна
- Если компания `SUSPENDED` → обе кнопки заблокированы + текст «Подписка не оплачена»

**3. Блок «Сотрудники сегодня»**

- Три счётчика: В офисе (зелёный) / Ушли (серый) / Нет (красный)
- Тап → переход на экран «Офис» (для WORKER — только чтение; для ADMIN — с управлением)

**4. Мой график (мини)**

- Строка из 7 дней (Пн–Вс) с временем начала смены под каждым
- Сегодняшний день выделен синим кружком
- Тап «Подробнее» → экран «Моё расписание»

> **Убрано с главной:** блок быстрых ссылок (Профиль / Дежурство / Новости), превью новостей

## 7. QR-СКАНЕР (Check-In / Check-Out)

Открывается по кнопке «Приход» / «Уход» на главной.

### 7.1 Экран сканирования

- Полноэкранная камера
- Прямоугольная рамка по центру с анимацией сканирования
- Текст: «Наведите камеру на QR-код офиса»
- Кнопка «Назад» (крестик сверху)

### 7.2 Верификация (двойная проверка на backend)

1. QR-токен = активный QR компании
2. IP устройства = офисная сеть компании

Приложение только сканирует и отправляет — проверку делает backend.

### 7.3 Результат прихода

- ✅ **ON_TIME** — «Приход отмечен ✓» + время + зелёная анимация
- ⚠️ **LATE** — «Опоздание на X мин» + время + жёлтый акцент
- 🕐 **EARLY_ARRIVAL** — «Пришли на X мин раньше» + время + синий акцент

### 7.4 Результат ухода

- ✅ **ON_TIME** — «Уход отмечен ✓» + время + отработанные часы
- ⬅️ **LEFT_EARLY** — «Ранний уход, осталось X мин» + время (если нет одобренной заявки)
- ⏱ **OVERTIME** — «Задержались на X мин» + время

### 7.5 Ошибки

| Ситуация | Сообщение |
| --- | --- |
| Неверный / просроченный QR | «QR недействителен. Попросите администратора обновить код.» |
| Вне офисной сети | «Вы не в офисной сети. Подключитесь к Wi-Fi офиса.» |
| Нет интернета | «Нет подключения. Проверьте интернет и попробуйте снова.» |
| Компания SUSPENDED | «Подписка не оплачена. Обратитесь к администратору.» |
| Уже отмечен сегодня | «Приход уже отмечен сегодня в HH:MM.» |

## 8. ЭКРАН «ОФИС» (Сейчас в офисе)

### WORKER (только чтение)

- Заголовок: «Сейчас в офисе» + количество
- Список: аватар + имя + время прихода
- Пустое состояние: «Никого ещё нет в офисе»
- Pull-to-refresh

### ADMIN (чтение + управление)

- То же, что у WORKER
- У каждого сотрудника в списке: кнопка «•••» → «Указать время ухода»
- Модальный экран: поле времени ухода → «Сохранить» → запись становится `MANUAL`

## 9. РАСПИСАНИЕ (Мой график)

- Шапка-карточка: суммарные часы в неделю + кол-во рабочих дней
- Список дней (Пн–Вс):
    - Иконка дня (Пн/Вт и т.д.)
    - Название + «Сегодня» бейдж если актуально
    - Время смены (start–end) или «Выходной» серым
    - Количество часов справа
    - Активный день — карточка с синей обводкой
- Только чтение (настройка на вебе)

## 11. ЗАЯВКИ

### 11.1 Две вкладки: «Новая заявка» и «Мои заявки»

### 11.2 Новая заявка

**Тип заявки** — выбор чипом (один активный):

| Тип | Описание |
| --- | --- |
| Больничный | Отсутствие по болезни |
| Семейные обстоятельства | Личная/семейная причина |
| Отпуск | Плановый отпуск |
| Командировка | Рабочая поездка |
| Удалённая работа | Работа не из офиса |
| Опоздание (по причине) | Объяснение опоздания |
| Ранний уход (по причине) | Запрос на уход раньше |
| Другое | Иная причина |

**Поля:**

- Дата начала (обязательно)
- Дата окончания (необязательно — для многодневных)
- Комментарий (textarea, необязательно)

**Кнопка:** «Отправить заявку» (активна только если тип + дата начала заполнены)

**Логика по типам:**

- `Больничный`, `Семейные обстоятельства`, `Отпуск`, `Командировка`, `Удалённая работа` → при одобрении дни помечаются `APPROVED_ABSENCE`
- `Ранний уход (по причине)` → при одобрении ранний check-out не штрафуется (`EARLY_LEAVE` approved)
- `Опоздание (по причине)` → объяснение опоздания, не меняет статус, служит для истории
- `Другое` → на усмотрение ADMIN

### 11.3 Мои заявки

- Список своих заявок: тип, дата, статус-чип (На рассмотрении / Одобрено / Отклонено)
- Тап → детали + комментарий ADMIN при отклонении

## 12. НОВОСТИ

### 12.1 Лента (оба)

- Карточки: тег «Новость», дата, заголовок, краткий текст, «Читать далее →»
- Непрочитанные — лёгкий жёлтый фон карточки
- Pull-to-refresh, пагинация

### 12.2 Детальная новость (оба)

- Тег, дата, крупный заголовок, полный текст, фото если есть
- При открытии → автоматически отправляется read-tracking (`news_reads`)
- Бейдж «✓ Отмечено как прочитанное» — появляется после отправки

### 12.3 Создание новости (только ADMIN)

- Кнопка «+» или FAB в правом нижнем углу ленты
- Поля: заголовок, текст, фото (опционально)
- «Опубликовать» → push всем активным сотрудникам компании

## 14. ПОДПИСКА И ОПЛАТА (только ADMIN)

Доступ: Профиль → «Подписка и оплата»

### 14.1 Экран статуса подписки

**Карточка статуса:**

- Статус компании: TRIAL / ACTIVE / GRACE / SUSPENDED (с цветовым индикатором)
- Тариф: $30 / месяц
- Дата следующего списания (если ACTIVE)
- Дней осталось в текущем периоде (если TRIAL или GRACE)

**Кнопки:**

- TRIAL / GRACE / SUSPENDED → **«Оплатить подписку»** (синяя, крупная)
- ACTIVE → **«История платежей»** + «Отменить подписку» (серый текст)

### 14.2 Экран оплаты

- Сумма: **$30.00**
- Краткое описание: «Подписка SoftTime на 1 месяц»
- Кнопка выбора способа оплаты (открывает платёжный провайдер — TBD)
- После успешной оплаты → статус обновляется до ACTIVE → экран «Оплата прошла ✓»

### 14.3 История платежей

- Список: дата, сумма, период, статус (Оплачено / Ошибка)
- Тап → детали платежа

## 17. СОСТОЯНИЯ И ОШИБКИ

Каждый экран с данными имеет 4 состояния:

| Состояние | Отображение |
| --- | --- |
| Loading | Skeleton-заглушки (не spinner поверх пустого экрана) |
| Empty | SVG-иллюстрация + текст + кнопка действия (где уместно) |
| Error | Иконка ошибки + текст + кнопка «Повторить» |
| Success | Данные |

**Офлайн:**

- Check-in / check-out, публикация новости, отправка заявки — **не выполняются без сети**, показывается баннер «Нет подключения»
- Чтение данных — показываются кэшированные (TanStack Query cache), с пометкой «Данные могли устареть»

## 18. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

- **Безопасность:** Secure Storage для токенов, весь трафик HTTPS, чувствительные данные не логируются
- **Производительность:** холодный старт < 3 с, списки через FlatList с getItemLayout
- **Tenant-изоляция:** клиент работает строго со своей компанией, разделение прав — на backend
- **Разрешения:** запрашивать камеру и push с объяснением («Нужно для сканирования QR»), корректная работа при отказе
- **Локализация:** русский по умолчанию, строки вынесены в i18n-файлы для будущих языков
