## 3. ДИЗАЙН-ТРЕБОВАНИЯ

### 3.1 Общие принципы

Цель — **строгая профессиональная админка**, без визуального шума.

- **Layout:** фиксированный левый сайдбар (навигация) + верхний топбар (название компании / меню пользователя) + контентная область с максимальной шириной ~1280px по центру.
- **Данные = таблицы.** Списки сотрудников, посещаемости, платежей — это таблицы с сортировкой, пагинацией и фильтрами, а не карточки.
- **Действия в модалках/шторках.** Создание, редактирование, подтверждение — в `Dialog` / `Sheet` shadcn, не на отдельных страницах где не нужно.
- **Пространство и иерархия.** Заголовок раздела + краткое описание + панель действий справа (кнопка «Добавить» и т.п.) + контент.
- **Статус-бейджи** едиными цветами (см. ниже).
- **Пустые состояния** — иконка + текст + кнопка действия.
- **Не делать:** разноцветные карточки, лишние градиенты, тяжёлые анимации, дашборды-«простыни» из десятков виджетов.

## 4. АУТЕНТИФИКАЦИЯ

### 4.1 Страница входа

- Лого + форма: email + пароль + кнопка «Войти»
- Под формой — кнопка **«Создать аккаунт (для администраторов)»**
- Ошибки: неверные данные / заблокирован
- После входа: ADMIN → дашборд компании, PROVIDER → дашборд платформы (по роли из токена)

### 4.2 Регистрация компании (только ADMIN)

- Доступна по кнопке со страницы входа
- Поля: название компании, ФИО владельца, email, пароль (≥ 8 символов)
- После создания → показать **код компании (6 символов)** с кнопкой «Скопировать» и пояснением «Передайте код сотрудникам»
- Компания создаётся со статусом `TRIAL` (первый месяц бесплатно)

> WORKER на вебе зарегистрироваться **не может** — формы для сотрудника нет.
> 

### 4.3 Токены

- Access 15 мин, Refresh 30 дней
- Access — в памяти, Refresh — httpOnly cookie (предпочтительно) или secure storage
- Перехватчик: 401 → silent refresh → при провале → выход на Login

## 5. ПАНЕЛЬ ADMIN

### 5.1 Сайдбар-навигация

```
Дашборд · Сотрудники · Посещаемость · Расписания · Заявки ·Новости · Офисные сети · QR-коды · Отчёты · Подписка · Аудит-лог · Настройки
```

Внизу сайдбара — название компании + код компании + меню пользователя (профиль, выход).

### 5.2 Дашборд

- Верхние карточки-метрики: сотрудников всего / сейчас в офисе / ожидают подтверждения / новых заявок
- Виджет «Сейчас в офисе»: список с именами и временем прихода + ручная отметка ухода для забытых
- Виджет статуса подписки: статус + дней до оплаты + кнопка «Оплатить» если TRIAL/GRACE/SUSPENDED
- График посещаемости за неделю (Recharts)

### 5.3 Сотрудники

- Таблица: ФИО, email, статус, дата найма, последняя активность
- Фильтр по статусу (PENDING / ACTIVE / WARNING / BLOCKED)
- Поиск по имени/email
- Действия по строке:
    - **PENDING** → «Принять» / «Отклонить»
    - изменение статуса (активировать / заблокировать)
    - открыть профиль сотрудника (история посещаемости, заявки, комментарий админа)
    - мягкое удаление (soft delete)
- Профиль сотрудника: данные + история посещаемости + заявки + поле Admin-комментария
- **Роль не меняется** (в компании один ADMIN)

### 5.4 Посещаемость

- Таблица по дням: сотрудник, дата, приход, уход, статус дня, отработано
- Фильтры: период (от–до), сотрудник, статус
- **Ручные правки:**
    - исправить время прихода/ухода (`is_manual = true`)
    - указать время ухода для забытого check-out
    - добавить запись об отсутствии с причиной (`note`)
- Все правки логируются в аудит-лог
- Раздел «Сейчас в офисе» — кто на месте сегодня + ручная отметка ухода

### 5.5 Расписания

- Список сотрудников с их недельными графиками
- **Два режима настройки:**
    - **Индивидуально** — открыть график конкретного сотрудника, редактировать по дням
    - **Один для всех** — создать шаблон графика и применить ко всем сотрудникам (или выбранным); у каждого остаётся редактируемая копия
- Редактор графика (по дням Пн–Вс):
    - `is_working_day` (чекбокс «Рабочий»)
    - `start_time` + `end_time`
    - `auto_checkout_buffer` — поле буфера автозакрытия рядом со временем (по умолчанию +1 ч)
    - Выходной → времена и буфер скрыты
- Валидация: минимум рабочего дня **6 часов** (ошибка при сохранении графика < 6 ч)

### 5.6 Заявки

- Таблица входящих заявок: сотрудник, тип, период, статус, дата подачи
- Типы: Больничный, Семейные обстоятельства, Отпуск, Командировка, Удалённая работа, Опоздание (по причине), Ранний уход (по причине), Другое
- Фильтр по статусу: На рассмотрении / Одобрено / Отклонено
- Действия: «Одобрить» / «Отклонить» (с комментарием)
- При одобрении отсутствия → дни `APPROVED_ABSENCE`; раннего ухода → не штрафуется

### 5.7 Новости

- Таблица/список новостей: заголовок, дата, «Прочитали N из M»
- Создать/редактировать новость: заголовок, текст, фото (опционально)
- Детально по новости: список кто прочитал / кто нет (read-tracking)
- Публикация → push всем активным сотрудникам компании

### 5.8 Офисные сети

- Таблица разрешённых IP/подсетей компании
- CRUD: добавить / редактировать / удалить (физическое удаление — это конфигурация)
- Для нескольких офисов — несколько сетей

### 5.9 QR-коды

- Список активных QR по офисам/сетям
- **Генерация / регенерация QR** (только ADMIN) — привязка к офисной сети
- Отображение QR на экране для печати/размещения в офисе
- При смене офиса/сети — регенерировать QR

### 5.10 Отчёты

- Фильтры: неделя / месяц / произвольный период / по сотруднику
- Показатели:
    - Кто работал больше всего (суммарные часы)
    - Кто приходил раньше / уходил позже
    - Кто больше пропустил без причины (ABSENT / INCOMPLETE)
    - Кто больше пропустил с причиной (APPROVED_ABSENCE)
    - Кто чаще опаздывал (LATE + кол-во)
    - Итоговая таблица по каждому сотруднику
- Экспорт в CSV/Excel
- Графики (Recharts): часы по сотрудникам, динамика опозданий

### 5.11 Подписка и оплата

- Карточка статуса: статус компании (TRIAL/ACTIVE/GRACE/SUSPENDED), тариф **$30/мес**, дата следующего списания, дней до конца периода
- Кнопка «Оплатить подписку» (для TRIAL/GRACE/SUSPENDED)
- История платежей: дата, сумма, период, статус
- Отмена подписки
- Платёжный шлюз — TBD (абстрактный слой: модуль оплаты + webhook)

### 5.12 Аудит-лог

- Таблица действий ADMIN: кто, что сделал, с чем (сущность), когда
- Фильтры: период, тип действия
- Только чтение

### 5.13 Настройки

- Общие настройки компании (рабочие часы по умолчанию, буфер автозакрытия по умолчанию)
- Профиль ADMIN: смена пароля
- Просмотр и копирование кода компании

## 6. ПАНЕЛЬ PROVIDER

### 6.1 Сайдбар-навигация

```
Дашборд платформы · Компании · Оплаты / Выручка
```

### 6.2 Дашборд платформы

- Метрики: всего компаний / активных / на триале / приостановленных
- Выручка: MRR (ежемесячный доход), суммарная выручка
- График роста компаний и выручки по месяцам
- Список последних регистраций и последних платежей

### 6.3 Компании

- Таблица: название, код, статус, число сотрудников, статус подписки, дата следующей оплаты, дата регистрации
- Фильтры: статус компании, статус подписки, поиск
- Действия: **активировать / приостановить** компанию вручную (например при споре по оплате)
- Тап по строке → детальная страница компании:
    - Информация о компании + подписка
    - **Сотрудники компании (только чтение)**
    - История платежей этой компании
- PROVIDER **не вмешивается** в операционные данные (посещаемость, расписания) — только надзор и биллинг

### 6.4 Оплаты / Выручка

- Таблица всех платежей по всем компаниям: компания, дата, сумма, период, статус, способ
- Фильтры: период, компания, статус платежа
- Сводка: суммарная выручка за период, кол-во платежей, средний чек
- Экспорт в CSV/Excel

## 8. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

- **Безопасность:** токены безопасно (httpOnly cookie / память), весь трафик HTTPS, разделение прав на backend (фронт не доверенная сторона)
- **Tenant-изоляция:** ADMIN никогда не видит чужие компании; проверка `company_id` на каждом запросе
- **Адаптивность:** оптимизировано под десктоп (1280px+), корректно сворачивается до планшета; мобильная вёрстка не обязательна (для телефона есть приложение)
- **Производительность:** таблицы с серверной пагинацией, lazy-загрузка разделов (code-splitting по роутам)
- **Локализация:** русский по умолчанию, строки в i18n
- **Доступность:** клавиатурная навигация по формам и таблицам, читаемый контраст
