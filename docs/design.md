# SoftTime — ДИЗАЙН-СИСТЕМА

# ТЕХНИЧЕСКОЕ ЗАДАНИЕ (Июнь 2026)

> Единая система для двух платформ: **React Native (мобайл)** и **React + shadcn/ui (веб)**.
Все токены (цвета, шрифт, отступы, радиусы) — общие. Реализация отличается только инструментами (Tailwind на вебе, StyleSheet на мобайле), но результат выглядит как один продукт.
> 

---

## 1. ФИЛОСОФИЯ ДИЗАЙНА

### Тон продукта

SoftTime — профессиональный инструмент для бизнеса. Дизайн должен передавать:

- **Доверие и надёжность** — всё работает, всё понятно, нет сюрпризов
- **Лаконичность** — только нужная информация, без лишнего
- **Уважение к времени** — пользователь открыл приложение чтобы быстро что-то сделать, не рассматривать интерфейс

### Чего НЕ делать

- Градиентные заливки на каждой карточке
- Анимации ради анимаций
- Разноцветные иллюстрации в стиле SaaS-лендингов
- Плотная «набивка» экрана блоками
- Несогласованные отступы (случайные 11px, 17px)
- Текст мельче 12px на мобайле

### Ключевой принцип

Если элемент не помогает пользователю принять решение или выполнить действие — его быть не должно.

### 2.3 Правила применения цветов

- Primary используется **только** для главного действия на экране и активных состояний навигации. Не красить им всё подряд.
- Тёмный текст (`color-text-primary`) — заголовки и важные данные. Серый (`color-text-secondary`) — пояснения, метки полей, вторичные данные.
- Красный (`color-danger`) — только реальные проблемы и деструктивные действия. Не для декора.
- Никаких цветных фонов секций, кроме белых карточек на сером фоне.

## 3. ТИПОГРАФИКА

### 3.1 Шрифт

**Семейство:** `Manrope` (Google Fonts)

- Полная поддержка кириллицы
- Геометрический, современный, читается при любом размере
- Ощущается профессионально, но не казённо

**Подключение:**

```css
/* Web */
@import url('https://fonts.googleapis.com/css2?family=Manrope:wght@400;500;600;700&display=swap');

/* Mobile */
// Установить через expo-font или react-native-google-fonts
```

### 3.2 Шкала текста (Type Scale)

| Токен | Размер | Вес | Line Height | Использование |
| --- | --- | --- | --- | --- |
| `text-xs` | 11px | 400 | 16px | Метки, временные метки |
| `text-sm` | 13px | 400 | 20px | Вспомогательный текст, подписи полей |
| `text-base` | 16px | 400 | 22px | Основной текст, тело |
| `text-base-medium` | 17px | 500 | 22px | Подзаголовки списков |
| `text-lg` | 18px | 600 | 24px | Заголовки карточек, модалок |
| `text-xl` | 20px | 700 | 28px | Заголовки экранов (мобайл) |
| `text-2xl` | 24px | 700 | 32px | Заголовки разделов (веб) |
| `text-3xl` | 30px | 700 | 38px | Крупные метрики, цифры в дашборде |

### 3.3 Правила типографики

- Не более **3 размеров** на одном экране
- Вес 400 (regular) = тело; 500 (medium) = подзаголовки; 600–700 (semibold/bold) = заголовки
- Цифры в карточках метрик — `text-3xl`, bold, `color-text-primary`
- Никаких капслоков кроме аббревиатур (QR, ADMIN и т.п.)
- Минимальный размер: **11px** — и только для меток, не для читаемого текста

### 7.5 Таблицы (только веб, TanStack Table)

- Заголовок: `text-xs` 600, `color-text-secondary`, uppercase, border-bottom `#E5E7EB`
- Строка: высота `52px`, border-bottom `1px solid #F3F4F6`, hover `#FAFAFA`
- Текст ячейки: `text-sm` 400
- Первый столбец: `text-sm` 500, `color-text-primary`
- Пагинация: кнопки Previous/Next + счётчик «1–20 из 156»

### 7.6 Модалки / Диалоги (веб)

- Overlay: `rgba(0,0,0,0.4)`
- Контейнер: `max-width: 480px`, `radius-lg`, `shadow-lg`, `bg: white`
- Структура: заголовок (text-lg) + описание (text-sm, secondary) + контент + кнопки
- Кнопки справа-внизу: ghost (cancel) + primary/danger (действие)

### 7.7 Bottom Sheet (мобайл)

- Выезжает снизу, `radius-xl` сверху, handle-индикатор (40×4px, серый, радиус full)
- Анимация: spring, 300ms
- Overlay закрывает при тапе

### 7.8 Toast / Уведомления

- Появляется снизу (мобайл) / сверху-справа (веб)
- Ширина: 320px (веб), 90% ширины экрана (мобайл)
- Варианты: success (зелёный), error (красный), info (синий), warning (жёлтый)
- Auto-dismiss: 3–4 сек
- Иконка слева + текст + опционально кнопка «Отменить»

### 7.9 Skeleton-заглушки

При загрузке данных — не spinner поверх пустого экрана, а скелетон:

- Прямоугольники с `radius-sm`, цвет `#E5E7EB`, анимация shimmer (градиент слева-направо)
- Для карточек: прямоугольник на месте заголовка + несколько строк
- Для таблицы (веб): строки с колонками-прямоугольниками

### 7.10 Пустые состояния

```
     [SVG иллюстрация]        ← простой монохромный SVG, 120×120px
   Нет данных за период
Сотрудники ещё не добавлены   ← text-sm, color-text-secondary, текст центр
   [ + Добавить сотрудника ]   ← кнопка secondary (если есть действие)
```

## 9. СИСТЕМА СТАТУСОВ

Единая визуальная система для любого статуса в любом контексте:

| Группа | Статусы | Цвет | Иконка |
| --- | --- | --- | --- |
| Успех / Норма | ON_TIME, PRESENT, ACTIVE, PAID | Зелёный | `CheckCircle2` |
| Внимание | LATE, GRACE, TRIAL, WARNING | Жёлтый | `AlertTriangle` |
| Проблема | ABSENT, INCOMPLETE, SUSPENDED, BLOCKED, REJECTED | Красный | `XCircle` |
| Одобрено | APPROVED_ABSENCE, APPROVED | Синий | `CheckCircle2` |
| Ожидание | PENDING | Синий | `Clock` |
| Нейтрально | EARLY_LEAVE, OVERTIME, MANUAL, LEFT_EARLY | Серый | `Info` |
| Ранний приход | EARLY_ARRIVAL | Синий | `LogIn` |

В каждом контексте (бейдж, строка таблицы, карточка):

- Всегда одновременно цвет + иконка + текст (не только цвет)
- Текст — человекочитаемый, не технический: «Вовремя», «Опоздание», «Не отмечен» (не `ON_TIME`, `LATE`)

## 11. АНИМАЦИИ И ПЕРЕХОДЫ

### Принципы

- Анимации функциональны: помогают понять что произошло, не развлекают
- Длительность: 150ms–300ms. Длиннее 400ms — только модалки/шторки
- Easing: `ease-out` (появление), `ease-in` (уход), `spring` (физика на мобайле)

### Конкретные анимации

| Элемент | Анимация | Длительность |
| --- | --- | --- |
| Переход между экранами (мобайл) | slide (right-in / left-out) | 250ms |
| Bottom Sheet | spring slide up | 300ms |
| Модалка (веб) | opacity + scale 0.96→1 | 200ms |
| Toast | slide + fade | 200ms |
| Кнопка pressed | scale 0.97 | 100ms |
| Skeleton shimmer | linear gradient loop | 1200ms |
| Бейдж появление | scale 0→1 | 150ms |
| Success экран (check-in) | bounce checkmark | 400ms |

### Чего не делать

- Сложные keyframe-анимации при каждом рендере
- Параллакс и 3D-трансформы
- Анимированные иконки (лоти) как основные элементы

## 12. АДАПТАЦИЯ ПЛАТФОРМ

Одни токены — разная реализация:

| Токен | Web (Tailwind) | Mobile (StyleSheet) |
| --- | --- | --- |
| `color-primary` | `text-[#1877F2]` / `bg-[#1877F2]` | `color: '#1877F2'` |
| `space-4` | `p-4` (16px) | `padding: 16` |
| `radius-lg` | `rounded-2xl` | `borderRadius: 16` |
| `shadow-sm` | `shadow-sm` | `elevation: 2` / iOS shadow |
| `text-xl` | `text-xl font-bold` | `fontSize: 20, fontWeight: '700'` |

**Ключевые различия платформ:**

|  | Mobile | Web |
| --- | --- | --- |
| Тени | elevation (Android) / shadow (iOS) или border | box-shadow через Tailwind |
| Шрифт | Manrope через expo-font | Manrope через Google Fonts CSS |
| Карточки | Чаще border вместо тени | Тень shadow-sm |
| Интерактивность | TouchableOpacity + Haptics | hover + cursor:pointer |
| Скролл | FlatList / ScrollView (нативный) | overflow-y auto |
| Поля ввода | h:48px минимум (палец) | h:40px |
| Кнопки | h:48–56px (палец) | h:36–44px |

## 13. ДОСТУПНОСТЬ

- Минимальный контраст текста: **4.5:1** (нормальный), **3:1** (крупный/bold)
- Минимальная область тапа (мобайл): **44×44px**
- Все иконки с семантическим `aria-label` или `accessibilityLabel`
- Поля формы: `label` связан с `input`
- Цвет не единственный индикатор статуса (всегда + текст или иконка)
